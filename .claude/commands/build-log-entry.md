# /build-log-entry — Build Log Entry

Append a single build-activity event to `.github/build-logs/progressiov1-build-log.csv`.

## Procedure

1. **Run recovery hash** (idempotent) to ensure the widest output CSV
   (`data/lake/enriched/master/normalized/gold_domestic_daily_panel.csv`)
   is quarantined by `scripts/pipeline/recovery_hash_backup.py` before any
   destructive operation on the build log or related output.

2. **Construct the entry JSONL line** with exactly these keys:

   ```json
   {"ts":"<ISO-8601>","source":"<pipeline/...>","action":"<started|completed|error>","detail":"<message>","worker_id":"<who>"}
   ```

   - `ts` — UTC ISO-8601 timestamp of the event.
   - `source` — fully-qualified path or script identifier of the origin.
   - `action` — one of: `started`, `completed`, `error`, `skipped`.
   - `detail` — free-text; keep under 300 chars.
   - `worker_id` — the agent or worker that generated this entry.

3. **Append the entry** using `scripts/pipeline/build_log_replay.py`:

   ```powershell
   python scripts/pipeline/build_log_replay.py --append-file event.jsonl
   ```

   This call:
   - Ensures the 5-line preamble is present in the CSV.
   - Computes the per-entry `body_hash` and `dedup_signature`.
   - Removes duplicate entries.
   - Groups rows by `source` so workers are "aligned" in the CSV.

4. **Report back** the dedup_signature and body_hash so the orchestrator
   can track the entry without re-reading the full CSV.

## 5-line preamble (hardcoded in build_log_replay.py)

```
## Build Log — ProgressIO v1
## Format: timestamp, source, action, detail, worker_id, dedup_signature, body_hash
## WARNING: do not edit this file manually. Use build_log_replay.py or //build-log-entry.
## Generated: <now>
## Valid range: 2010-01-01 → 2027-12-31
```

## Dedup logic

Duplicate = same `(source, ts, action, body_hash)`.  If a matching entry
already exists (either in the file or in the in-progress replay session),
it is silently dropped and reported as `[dup]`.

## Restore from backup

If the build log was accidentally wiped, restore the widest output CSV
from the quarantine backup:

```powershell
$m = Get-Content .github/build-logs/backups/quarantine_manifest.json | ConvertFrom-Json
Copy-Item $m.backup_file $m.source_file
```
